# Security Policy

## Supported Versions

This project is a static site currently maintained on the main branch only. Security updates are applied to the latest stable version.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Scope

This security policy covers:

- The static site generated by Eleventy (HTML, CSS, JavaScript).
- Form handlers and client-side validation (contact form, Starter Kit lead capture).
- Webhook integrations (e.g., Make) used for lead submission.
- npm dependencies and build tooling.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly using one of the following methods:

- **Public issues (non-sensitive bugs):** Open a [GitHub Issue](https://github.com/rfonte/brazildecoded.webapp/issues) with the `security` label.
- **Sensitive vulnerabilities:** Send an email to the maintainers with a clear description, steps to reproduce, and potential impact.

### What to expect

- **Acknowledgment:** Within 48 hours of receiving your report.
- **Status update:** Within 7 days, with an assessment of the vulnerability and planned remediation.
- **Resolution:** Once fixed, we will publish a summary (for public issues) or notify you directly (for private disclosures).
- **Credit:** Reporters will be credited in the release notes unless they wish to remain anonymous.

If a report is declined, we will provide a clear explanation of our reasoning.

## Data Protection

This site collects personal data (name, email, phone) through forms for the Starter Kit and contact pages. We are committed to:

- Obtaining **explicit consent** before any data submission (via a required consent checkbox).
- Processing data in compliance with **LGPD** (Lei Geral de Proteção de Dados) and applicable privacy regulations.
- Not storing or transmitting personal data without valid consent.

If you identify any issue related to data handling, consent mechanisms, or third-party integrations, please report it as described above.

## Dependency Security

We monitor npm dependencies for known vulnerabilities. To check the current state locally:

```bash
npm audit
```

Security patches are applied as part of regular maintenance. If you are running a fork of this project, ensure you keep dependencies up to date.
